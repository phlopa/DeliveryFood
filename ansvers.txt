PR5_4
Контрольні питання

1. Розуміння концепції
Слайдер у веб-розробці: Це інтерактивний компонент для циклічного відображення контенту (зображень, карток, тексту) у обмеженому просторі. Використовується для акцентування уваги на головних пропозиціях та економії місця на сторінці.

Переваги Swiper.js:

Оптимізація під тач-пристрої: Найкраща обробка свайпів та жестів на мобільних телефонах.

Модульність: Можливість підключати лише необхідні функції (навігацію, пагінацію тощо), що зменшує вагу коду.

Велика екосистема: Наявність готових ефектів (Fade, Cube, Flip) та підтримка адаптивності "з коробки".

Ключові елементи структури:

.swiper — основний контейнер-оболонка.

.swiper-wrapper — внутрішній контейнер, що рухається.

.swiper-slide — окремий елемент (слайд) із контентом.

2. Функціональність
Типи слайдерів: Swiper дозволяє створювати горизонтальні (стандарт), вертикальні (параметр direction: 'vertical') та багаторядкові слайдери (параметр grid).

Автоматичне перелистування: Реалізується за допомогою модуля Autoplay.

Параметр: autoplay: { delay: 3000, disableOnInteraction: false }.

Кнопки навігації: Додаються через модуль Navigation. В HTML створюються блоки з класами .swiper-button-next та .swiper-button-prev.

Пагінація: Реалізується через модуль Pagination. Потребує наявності контейнера .swiper-pagination в HTML.

Ефекти переходу: Налаштовуються параметром effect. Доступні варіанти: 'fade', 'cube', 'coverflow', 'flip', 'cards'.

3. Адаптивність
Метод реалізації: Слайдер стає адаптивним завдяки параметру breakpoints. Він працює аналогічно до Media Queries у CSS.

Параметри конфігурації:

slidesPerView — кількість слайдів, що відображаються одночасно.

spaceBetween — відстань між слайдами у пікселях.

slidesPerGroup — кількість слайдів, які прокручуються за один раз.

4. Взаємодія
Методи взаємодії: Користувач може взаємодіяти зі слайдером через дотики (тап), просування пальцем/мишею (свайп) або за допомогою коліщатка миші (модуль mousewheel).

Додаткові елементи керування: Окрім кнопок, можна додати прогресбар (тип пагінації type: 'progressbar') або скролбар (клас .swiper-scrollbar).

5. Розширення функціональності
Підключення модулів: У сучасних версіях модулі імпортуються окремо для зменшення розміру бандлу. Приклади: Autoplay, Pagination, Navigation, EffectFade, Keyboard.

Власний модуль: Створюється шляхом написання функції, яка використовує внутрішні події Swiper (наприклад, on: { init: () => { ... } }) або розширює прототип класу Swiper.

6. Оптимізація
Способи оптимізації:

Lazy Loading: Завантаження зображень лише при наближенні слайда до зони видимості.

Virtual Slides: Рендеринг лише активних слайдів (важливо для списків із 100+ елементів).

Модульне підключення: Відмова від повної бібліотеки swiper-bundle.js на користь окремих модулів.

Фактори швидкості: Розмір (вага) зображень, кількість оброблюваних DOM-елементів, використання складних 3D-ефектів (cube, flip) та загальна потужність пристрою користувача.
